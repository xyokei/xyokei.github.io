一、理论

1.1 思想

  1. 分布式版本控制系统
  2. 对待数据的方式直接记录快照，而非差异比较

1.2 工作原理

  1. git 更新：对更新时点全部文件创建快照并保存快照的索引
  2. 本地执行
  3. 通过计算校验和（哈希散列机制）来保存数据的完整性
  4. git 一般只会添加数据（撤销操作不常用）
  5. git 项目三个阶段 工作区 索引 git 目录
    1. 工作区 ->索引
    2. 索引->git 目录
    3. git 目录->工作区
